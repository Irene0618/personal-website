# Xinwen Personal Website 网页框架文档

## 1. 网站定位

网站名称：The Growing Tree

核心概念：这不是一份静态简历，而是一座持续生长的数字花园。网站用“复古浏览器 + 作品拼贴 + 树木生长”的视觉语言，把你的研究生作品、本科影像作品、AIGC项目、产品原型、设计研究、摄影和内容运营经历组织成一棵不断生长的树。

核心一句话：

> Every project is a new ring in the trunk.

中文气质：

> 每一次创作，都是树干上的一道年轮。

## 2. 当前技术结构

项目类型：Vite + React + TypeScript + Tailwind CSS

主要文件：

- `src/App.tsx`：页面结构、内容数据、组件逻辑
- `src/index.css`：整体视觉风格、响应式布局、动画
- `public/portfolio/`：网页使用的作品封面图
- `public/portfolio/videos/`：网页使用的轻量演示视频
- `index.html`：网页入口
- `package.json`：项目脚本和依赖

运行命令：

```bash
npm run dev
```

如果 PowerShell 拦截 npm，可以使用：

```bash
npm.cmd run dev
```

构建命令：

```bash
npm.cmd run build
```

## 3. 页面信息架构

### 3.1 Hero / 首页首屏

模块 ID：`#top`

视觉参考：复古网页拼贴、纸张、照片、作品封面、自然树影。

当前元素：

- 《植系乌托邦》作为首屏背景图
- 中央 Polaroid 使用个人照片
- 左右辅助照片使用《思茧成蝶》《电子新乐府》项目封面
- 主标题：The Growing Tree
- 身份描述：AI Product / AIGC Operations / Design Research / Photography
- CTA按钮：View Featured Works

后续可修改点：

- 替换中央照片为更适合作品集首页的形象照
- 替换左右拼贴图为更想主推的项目
- 增加鼠标视差、纸张拖拽、叶片飘动等微交互

### 3.2 Roots / 根系

模块 ID：`#roots`

功能：展示作品背后的方法来源。

当前根节点：

- 用户洞察
- 影像叙事
- AIGC工作流
- 产品原型
- 内容运营
- 摄影观察
- 互动媒体
- 设计研究

后续可扩展：

- 每个根节点点击后展开一段方法论说明
- 加入书籍、理论、研究问题、项目决策依据
- 和具体项目建立关联

### 3.3 Trunk / 树干

模块 ID：`#trunk`

功能：About Me + 年轮式经历。

当前经历：

- 2020 Animation & Game Design
- 2022 Campus Content Operations
- 2024 Design Research
- 2025 Xinhua News Agency
- Now AI Product & AIGC Workflow

注意：

不要写“腾讯实习”。当前版本使用的是新华社音视频部经历。

### 3.4 Featured Works / 精选项目

模块 ID：`#branches`

功能：展示主推项目，每个项目后续可以扩展成独立 Case Study 页面。

当前项目：

1. The Cocoon Turns Into A Butterfly / 《思茧成蝶》AIGC实验短片
2. Electronic Yuefu / 《电子新乐府》AI诗词音乐实验室
3. Qin Yun / 《琴韵》五音疗愈音箱及配套APP
4. LadyWell / 《她好 LadyWell》更年期女性关爱平台
5. Botopia / 《植系乌托邦》自然关系思辨设计
6. Zhuangyuan Qijing / 《状元七景》长虹状元文化包装设计
7. Art Heals The Heart / 《以艺疗心》大学生艺术疗愈内容社区
8. Digital ZUEL / 数字中南大 - 元宇宙校园平台

推荐 Case Study 模板：

```text
Project Overview
Problem
Research
Ideation
Design
Iteration
Outcome
Reflection
```

当前已实现：
- 每张 Featured Works 卡片底部都有 `View Case Study` 入口。
- 页面地址格式为 `#case-项目slug`，例如 `#case-electronic-yuefu`。
- 8 个主推项目都已经有独立 Case Study 页面。

当前 Case Study 页面结构：
- Hero：项目类型、年份、英文标题、中文标题、项目概览、主视觉图。
- Process：项目流程，用编号步骤展示从概念到产出的过程。
- My Role：说明你在项目中负责的部分。
- Reflection：说明这个项目里学到的能力和方法。
- References：附上项目参考图、封面图、界面图或展示板。

当前图片展示规则：
- Featured Works 卡片采用“上图下文”结构。
- 卡片图片使用 `object-fit: contain`，优先保证作品封面、展示板和界面截图完整可读。
- 如果某个项目更适合强视觉封面，可以后续单独改成 `cover` 并设置图片焦点。

后续可修改：

- 继续精修每个 Case Study 的正文，使其更像正式求职作品集。
- 给每个 Case Study 增加更多过程图、草图、原型图、用户旅程图、Prompt 迭代图。
- 增加项目筛选：AI Product / AIGC Film / UX Research / Cultural Creative / Motion
